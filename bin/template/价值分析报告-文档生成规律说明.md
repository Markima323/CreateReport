# 价值分析报告文档生成规律说明

## 1. 分析范围

对比了 1 份原始模板、5 份已完成报告和 1 份 Excel 数据表。成品之间的差异可归为五类：项目级常量、Excel 直接映射、格式化派生值、条件性章节、计算及语义判断。

## 2. 关键结论

- 这不是单纯的“黄色文字全部替换”任务。模板中存在完整章节删除、同级标题重编号、债务人数量分支、安置费用二选一、结论最低值名称判断和身份证校验。
- 文件名使用支行与户名；报告编号使用序号。
- 正文预计清收日期使用 Excel 的“预计回收”列，不使用“清收完成时间”列。
- 债权评估值取“基准日本息、权利价值、可回收金额”三者最低值，结论中的名称必须跟随实际最低项变化。
- 安置费用为 0/空时，公式和计算式都不能出现安置费用或 `-0.00`。
- 保证人为空时，应删除“债务责任关联方简介”整个章节，并对后续章节编号前移。
- 债务人只有一人时写“债务人”；多人时按“债务人一、债务人二……”编号。
- 地址末尾室号用于推导楼层，例如 406 室为 4 层，3-402 室为 4 层。
- Excel 中段红宇身份证号为 17 位，成品按居民身份证校验位补为 18 位，说明生成程序需要身份证校验与受控修复。

## 3. 数据层级

### 项目级参数

项目年、报告年、报告批次、分析基准日、报告日期、报告日期大写、有效期截止日和折现率不应散落在 Prompt 中，应作为每次任务的显式参数传入。有效期截止日不能从现有样例可靠推导，应由调用方给定。

### 行级字段

每个“序号”非空行生成一份报告。直接字段包括支行、姓名、本金、利息、本息、地址、面积、权利价值、预计回收、单价、各项计算结果、权证编号、证载权利人、债务人和保证人。

### 派生字段

包括万元金额、百分比、民族显示、债务人称谓与连接字符串、楼层、安置费用段落、回收金额公式文字、最低值名称、文件名和报告编号。

## 4. 计算关系

- 基准日本息 = 基准日本金 + 基准日利息
- 评估价值 = ROUND(面积 × 评估单价, 0)
- 快速变现价值 = ROUND(评估价值 × 快速变现系数, 0)
- 折现系数 = ROUND((1 + 折现率)^(-折现期限), 4)
- 折现价值 = ROUND(快速变现价值 × 折现系数, 0)
- 可回收金额 = 折现价值 - 处置费用 - 安置费用
- 债权评估值 = MIN(基准日本息, 权利价值, 可回收金额)
- 偿债率 = 债权评估值 ÷ 基准日本息

Excel 已计算值作为报告值，同时由程序独立复核；不一致时输出警告，不宜让模型静默覆盖。

## 5. 推荐程序结构

1. 读取 Excel 并标准化每行数据。
2. 进行公式、身份证、必要字段和地址楼层校验。
3. 按 [Gemini 3.5 Flash 官方指南](https://ai.google.dev/gemini-api/docs/whats-new-gemini-3.5?hl=zh-cn) 调用 Interactions API 完成生成计划审核和可选的保证人公开信息检索，要求返回严格 JSON；Excel 明确字段优先，模型不得覆盖。检索来源只读取 API 返回的 `url_citation` 注解，不信任模型自行填写的网址。
4. 由确定性代码修改 DOCX 的 OOXML，而不是让模型重新排版文档。
5. 删除可选段落/表格行，更新域、目录和页码。
6. 对生成文件执行 XML 扫描、数值复核和页面渲染检查。

## 6. 模板说明

提供的基底模板沿用原模板版式，黄色高亮为普通替换字段，蓝色高亮为条件性字段。生成完成后必须移除全部高亮和未使用段落。模板中加入了债务人 3、债务人 4 的可选段落。程序通过 Microsoft Word 后台更新目录和页码，并保留 `TOC`、`PAGEREF`、`PAGE` 等域代码；模板不设置“打开文档时自动更新域”，避免 Word 弹出外部字段更新提示。
