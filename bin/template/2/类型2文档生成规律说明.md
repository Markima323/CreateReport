# 类型 2：企业价值评估文档生成规律

## 一、输出文件

类型 2 每个项目生成两份 Word：

1. `1-评估报告-<被评估单位>-<评估基准日>.docx`
2. `2-评估说明-<被评估单位>-<评估基准日>.docx`

评估报告需要生成或替换：

- 评估对象和评估范围
- 资产基础法具体评估方法介绍
- 评估结论

评估说明需要生成或替换：

- 第一章中的评估对象与评估范围说明
- 实物资产的分布情况及特点
- 第三章：资产基础法评估技术说明

生成流程为：先读取 Excel 形成类型 2 JSON，再调用 Gemini 生成动态章节初稿，最后由程序根据 Excel JSON 和 `资产基础法评估方法库.json` 进行对象报告式补全。补全后的 `generated_sections` 才用于写入 Word；Gemini 原始返回保存在 `gemini_generated_sections`，便于排查。

## 二、输入目录

面板输入根目录固定为 `Input`，选择类型 2 后程序自动读取 `Input/2`。`Input/2` 下每个直接子文件夹代表一份独立的评估项目，例如：

```text
Input/
└─ 2/
   ├─ 项目A/
   │  ├─ 3-评估明细表-项目A.xls
   │  └─ 成本法评估过程/
   └─ 项目B/
      ├─ 3-评估明细表-项目B.xls
      └─ 成本法评估过程/
```

每个完整评估项目文件夹可包含：

- 一份或多份名称含“评估明细表”的 `.xls`、`.xlsx`、`.xlsm` 文件；
- 成本法、资产基础法、收益法、市场法等评估工作底稿；
- 存货、房屋、构筑物、机器设备、车辆、电子设备、土地、专利等专项底稿。

忽略 `~$` 开头的 Office 临时文件。

若存在多份评估明细表，通常表示母公司及子公司。报告的“评估对象和评估范围”只使用被评估单位对应的母公司明细表。优先级如下：

1. `汇总-资产基础法` 中“被评估单位”与项目主体名称一致；
2. 文件名序号为 `3-` 或 `3.1-`；
3. 文件名中的单位简称与“被评估单位”一致；
4. 无法唯一确定时停止生成，不得自动合并母子公司明细。

## 三、Excel 读取规则

1. 只读取可见工作表，隐藏工作表不参与报告生成。
2. `汇总-资产基础法` 的金额单位是人民币万元。
3. `2分类汇总` 和其他明细表的金额单位通常是人民币元；仍应读取表内“金额单位”文字确认。
4. 科目名称不能依赖固定工作表编号，应去掉工作表名前面的数字编号后识别。
5. 表头可能为多行或合并单元格，应按表头文字识别字段，不能只按固定列号读取。
6. `汇总-资产基础法` 提供报告首页范围表、资产基础法结论和汇总表。
7. `2分类汇总` 提供各科目的账面价值、评估价值、增减值和增值率。
8. 各科目明细表提供项目数量、名称、业务内容、账龄、存放地点、权属、减值准备及其他描述信息。
9. 各可见科目明细表会完整抽取为 `subjects[].detail_table`。该完整表格不发送给 Gemini，只用于本地 JSON 和 Word 渲染。
10. Word 中默认只插入对象报告通常列示的小型明细表，例如长期股权投资、房屋建筑物、土地使用权；存货、机器设备、往来款等大表只保存在 JSON 中，避免生成文档远大于对象报告。可在 `detail_tables.report_scope_subjects` 和 `detail_tables.explanation_technical_subjects` 中调整。
11. 完整表格保留全部非空数据行，包括合计、小计等行；日期列会转换为中文日期，金额/价值/原值/净值/面积/数量等列保留两位小数。
12. 账号、号码、编号、证号、权证等长编号列应在源 Excel 中设置为文本格式；如果源 Excel 已用超长数字存储，Excel 本身可能已丢失精度，程序无法恢复被截断或四舍五入的数字。

## 四、专项底稿表格读取规则

1. `workpapers[].tables` 专门保存从成本法评估过程、成本法计算过程、收益法评估过程、市场法/交易案例比较法底稿中抽取的案例表、参数表、修正系数表、取费表、指数表、折旧/摊销表等。
2. 程序只抽取命中关键词且行列数受控的小型表格，避免把整本底稿、资产负债表或大型资产明细表写入第三章。
3. 底稿表格完整内容不发送给 Gemini；AI 输入中只保留表名、来源、行列数、关键词和科目提示，Word 插入由本地程序完成。
4. 插入位置按科目匹配：土地相关表进入土地使用权；房屋、构筑物、租金、重置价相关表进入对应实物资产科目；交易案例、对比公司、企业价值相关表进入长期股权投资。
5. 单元格内容优先读取 Excel 显示文本，用于保留源表中的百分比、日期、金额格式。

## 五、评估对象和评估范围

1. 评估对象通常为“被评估单位于评估基准日的股东全部权益价值”。
2. 评估范围为评估基准日资产负债表列示的全部资产和负债，以及企业申报的表外资产。
3. 首页范围表固定列示：
   - 流动资产
   - 非流动资产
   - 非流动资产中账面价值非零的各科目
   - 资产总计
   - 流动负债
   - 非流动负债
   - 负债总计
   - 净资产或股东全部权益价值
4. “资产及负债包括……”只列示账面价值不为零的四大类别：流动资产、非流动资产、流动负债、非流动负债。
5. 每个类别下只列示账面价值不为零的科目。
6. 某科目账面价值为零但评估价值大于零时，不放入“委估主要资产及负债情况”。
7. 技术类无形资产有内容但账面价值为零时，放入“企业申报的表外资产情况”。
8. 没有表外资产时使用固定文字：“本次评估企业无申报的表外资产。”

## 六、科目描述规则

1. 项目数量取明细表“序号”的最大有效整数，不取 Excel 有效行数。
2. 普通科目写“账面价值××万元”。
3. 存在减值准备字段且准备金额为零时，写“未计提坏账准备”或“未计提存货跌价准备”。
4. 减值准备金额不为零时，写“计提坏账准备××万元”或“计提存货跌价准备××万元”。
5. 应收、预付、应付等往来科目优先从“业务内容”归纳用途，并从账龄列归纳主要账龄。
6. 存货、设备、无形资产等优先从“名称”“名称及规格型号”“设备名称”等字段归纳内容。
7. 类型较少时完整列示；类型较多时优先选择出现次数多且金额合计较大的类型。
8. 不得使用工作表空白行、合计行或隐藏行推断项目数量。

## 七、实物资产分布情况

实物资产通常包括：

- 存货；
- 房屋建筑物、构筑物及其他辅助设施；
- 机器设备、车辆、电子设备；
- 在建工程；
- 土地使用权；
- 其他具有实物形态的资产。

每类资产按明细表及专项工作底稿描述数量、主要类型、面积、存放地点、取得方式、结构、使用状态和权属情况。没有可靠来源的地点、状态、结构和权属信息必须留空并记录警告。

## 八、资产基础法具体评估方法

1. 按报告中出现的科目顺序生成。
2. 只生成账面价值非零或评估价值非零的科目。
3. 方法文字优先参考已完成报告采用的表述。
4. 已完成报告没有对应科目时，使用 `资产基础法评估方法库.json`。
5. 房屋、土地、技术类无形资产等存在多种方法时，必须根据专项工作底稿中实际采用的方法选择，不能任意套用。
6. 报告中的“具体评估方法介绍”使用概括性方法文字；评估说明第三章使用包含评估范围、资产概况、评估过程、评估方法和评估结果的详细技术说明。

## 九、评估结论

1. 资产基础法结论完全取自 `汇总-资产基础法`，不得由模型重新计算后覆盖源表。
2. 增减值校验：`评估价值 - 账面价值 = 增减值`。
3. 账面价值不为零时，增值率校验：`增减值 / 账面价值 = 增值率`。
4. 股东全部权益校验：`资产总计 - 负债总计 = 股东全部权益价值`。
5. 最终结论采用哪一种评估方法，应由项目资料明确提供；无法确认时只输出各方法结果，不擅自选择最终方法。
6. 大写金额由程序根据最终评估值转换，不能由模型自由书写。

## 十、设备评估扩展规则

设备询价是独立扩展步骤，不应阻塞报告主体生成：

1. 抽取约 10% 的代表性设备进行市场询价。
2. 名称和规格型号相似度优先于价格接近度。
3. 参考未折扣价格应尽量接近账面原值乘以 1.13。
4. 启用日期越早，选取价格通常越低；计算机、显示器、服务器等电子设备折旧更快。
5. 同类名称和型号可共用一个可靠价格来源。
6. 找到市场价格时，“取价依据”填 URL，“购置单价（含税价）”填原始报价。
7. 未询价或未找到价格时，“取价依据”填“价格指数”，按通用设备、专用设备、电子设备、家具分类选取评估基准日前最近一期价格指数。
8. 评估基准日前六个月内启用的设备，“取价依据”填“账面价值”。
9. 经济寿命年限优先使用 `统一年限.xlsx`；电子设备通常为 8 年，通用机器设备通常为 12 年，专项底稿有明确年限时以专项底稿为准。

## 十一、禁止事项

- 不得把范例中的单位名称、金额、日期、地址、权证号或人员信息写入新项目。
- 不得把子公司的数据混入母公司“评估对象和评估范围”。
- 不得读取隐藏工作表来补齐可见表中不存在的数据。
- 不得猜测最终评估方法、产权瑕疵、抵押状态、审计状态和表外资产。
- 图片、Excel、Word 样例和生成 JSON 均属于敏感数据，不应提交到 Git。
